## What is HEALPix?

**HEALPix** (Hierarchical Equal Area isoLatitude Pixelization) is a standard for partitioning a sphere (like a planet or the sky) into pixels of equal surface area.

Unlike traditional "rectangular" map projections (like Equirectangular or Mercator), HEALPix ensures that:

<CardGroup cols={3}>
  <Card title="Equal Area" icon="grid-2">
    Every pixel is the same size, ensuring statistical analysis remains valid across the entire globe, including the poles.
  </Card>
  <Card title="Hierarchical" icon="sitemap">
    You can easily increase or decrease resolution (nside) while maintaining spatial relationships.
  </Card>
  <Card title="Fast Computation" icon="bolt">
    Efficient neighbor searches and spherical harmonic transforms.
  </Card>
</CardGroup>

## The Problem: Data Distortion & Scale

In planetary science, data often arrives as **scattered points, tracks, or footprints** from spectrometers and altimeters. Traditionally, researchers face two major hurdles:

<Note>
  **Projection Bias:** Standard grids distort the poles, making global surface calculations (like mean chemical abundance or crater density) mathematically biased.
</Note>

<Note>
  **The Memory Wall:** Modern missions generate billions of points. Loading an entire global high-resolution map into RAM to update it is often impossible.
</Note>

HEALPyxel solves this by treating the sphere as a **modern data engineering target** rather than just a geometric grid.

## Understanding nside Values

The `nside` parameter controls HEALPix resolution. It must be a **power of 2** (1, 2, 4, 8, 16, 32, 64, 128, ...).

```python
from healpyxel.core import validate_nside

# Valid nside values
nside = 64  # ✓ Valid (power of 2)
nside = 128 # ✓ Valid

# Invalid nside
nside = 100 # ✗ Invalid (not power of 2)
# Raises: ValueError: nside must be a power of 2
```

### Number of Cells

The total number of HEALPix cells for a given nside is:

```
N_pixels = 12 × nside²
```

**Examples:**
- nside=32 → 12,288 cells
- nside=64 → 49,152 cells
- nside=128 → 196,608 cells

## Nested vs Ring Ordering

HEALPix supports two cell ordering schemes:

- **Nested** (default in HEALPyxel): Nearby cells have nearby indices, enabling efficient spatial queries
- **Ring**: Cells ordered by latitude rings, useful for spherical harmonics

HEALPyxel uses **nested ordering** by default for optimal spatial locality.

```python
from healpyxel.sidecar import compute_healpix_ids_from_lonlat
import numpy as np

# Compute HEALPix cell IDs from coordinates
lons = np.array([0.0, 45.0, 90.0])
lats = np.array([0.0, 30.0, -30.0])
nside = 64

# Returns nested order indices by default
healpix_ids = compute_healpix_ids_from_lonlat(nside, lons, lats)
print(healpix_ids)  # e.g., [16384, 18234, 20156]
```
