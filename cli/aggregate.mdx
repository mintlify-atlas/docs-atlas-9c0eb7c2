---
title: healpyxel_aggregate
description: Aggregate spatial data by HEALPix cells using sidecar mappings
---

The `healpyxel_aggregate` command computes statistics for each HEALPix cell by grouping observations mapped in sidecar files. This implements the **APPLY + COMBINE** stages of the split-apply-combine pattern.

## Overview

Aggregate joins sidecar mappings with original data to compute cell-level statistics (mean, median, robust standard deviation, etc.) for specified value columns.

## Basic Usage

```bash
healpyxel_aggregate \
  --input observations.parquet \
  --sidecar-index 0 \
  --aggregate \
  --columns r750 r950 \
  --aggs median robust_std
```

## Required Arguments

<ParamField path="--input" type="path" required>
  Input parquet file with observational data. Must contain `source_id` column and value columns for aggregation.
  
  **Example:**
  ```bash
  -i observations.parquet
  ```
</ParamField>

<ParamField path="--sidecar-index" type="int | string" required>
  Select which sidecar(s) to process.
  
  **Options:**
  - `all`: Process all sidecars in batch
  - `0 2 4`: Specific sidecar indices (space-separated)
  - `0`: Single sidecar
  
  **Example:**
  ```bash
  --sidecar-index all  # Batch mode
  --sidecar-index 0    # Single sidecar
  ```
</ParamField>

<ParamField path="--aggregate" type="flag" required>
  Enable aggregation processing (safety flag to prevent accidental execution).
  
  **Example:**
  ```bash
  --aggregate
  ```
</ParamField>

<ParamField path="--columns" type="string[]" required>
  Value columns to aggregate (space-separated).
  
  **Example:**
  ```bash
  --columns reflectance radiance emission
  ```
</ParamField>

## Aggregation Functions

<ParamField path="--aggs" type="choice[]" default="mean median std robust_std">
  Statistical functions to compute per cell.
  
  **Available functions:**
  - `mean`: Arithmetic mean
  - `median`: Median (50th percentile)
  - `std`: Standard deviation
  - `min`: Minimum value
  - `max`: Maximum value
  - `mad`: Median Absolute Deviation (robust to outliers)
  - `robust_std`: MAD × 1.4826 (equivalent to σ for normal distributions)
  
  **Output columns:** Each function produces `{column}_{agg}` (e.g., `r750_median`, `r750_robust_std`)
  
  **Example:**
  ```bash
  --aggs mean median mad robust_std
  ```
</ParamField>

<Note>
**Robust statistics** (`mad`, `robust_std`) are recommended for planetary science data:
- Less sensitive to outliers and cosmic ray hits
- More stable with small sample sizes
- Better handles non-Gaussian distributions
</Note>

## Optional Arguments

### Input/Output Control

<ParamField path="--sidecar-dir" type="path">
  Directory containing sidecar files. Defaults to input file directory.
  
  **Example:**
  ```bash
  --sidecar-dir sidecars/
  ```
</ParamField>

<ParamField path="--output" type="path">
  Output file or directory. Auto-generated by default.
  
  **Auto-naming pattern:**
  ```
  {input_stem}-aggregated[-densified].{sidecar_suffix}.parquet
  ```
  
  **Example:**
  ```bash
  --output results/aggregated.parquet
  ```
</ParamField>

### Quality Control

<ParamField path="--min-count" type="int" default="1">
  Minimum observations per cell for valid statistics. Cells below threshold output NaN.
  
  **Recommended values:**
  - `1`: Include any cell touched by data (permissive)
  - `2-3`: Recommended for robust_std/MAD (handle outliers)
  - `5+`: Strict validation (reject sparse cells)
  
  **Example:**
  ```bash
  --min-count 3
  ```
</ParamField>

<ParamField path="--filter" type="string">
  Pandas query expression to filter observations before aggregation.
  
  **Example:**
  ```bash
  --filter "(qa_flag > 0.8) and (wavelength < 1000)"
  ```
</ParamField>

<Warning>
Filtering happens **before** aggregation, so `--min-count` applies to filtered data only.
</Warning>

### Densification

<ParamField path="--densify" type="flag">
  Fill all HEALPix cells (0 to 12×nside²−1), setting unobserved cells to NaN.
  
  **Trade-offs:**
  - **Sparse** (default): Smaller files, only observed cells
  - **Dense**: Uniform grids for visualization, larger files
  
  **Example:**
  ```bash
  --densify  # Create full grid
  ```
</ParamField>

### Data Loading Backends

<ParamField path="--use-duckdb" type="flag" default="true">
  Use DuckDB for efficient column projection and filtering (enabled by default).
  
  **Benefits:**
  - Reduces memory usage for wide tables
  - Pushes filters to storage layer
  - Recommended for files >1GB
  
  **Example:**
  ```bash
  --use-duckdb  # Explicitly enable (default)
  ```
</ParamField>

<ParamField path="--no-duckdb" type="flag">
  Disable DuckDB and use pandas/dask instead.
  
  **Example:**
  ```bash
  --no-duckdb  # Fallback to pandas
  ```
</ParamField>

<ParamField path="--use-dask" type="flag">
  Enable Dask for parallel aggregation.
  
  **Benefits:**
  - Parallel groupby operations
  - Lower memory footprint
  - Best for datasets >1M rows
  
  **Example:**
  ```bash
  --use-dask --dask-npartitions 8
  ```
</ParamField>

<ParamField path="--dask-npartitions" type="int">
  Number of Dask partitions for parallel processing. Defaults to CPU count - 1.
  
  **Tuning guidance:**
  - N = CPU cores: Good I/O balance
  - N = 2 × CPU cores: I/O-bound workloads
  - N = 1: Debug mode (no parallelism)
  
  **Example:**
  ```bash
  --dask-npartitions 16
  ```
</ParamField>

## Output Files

For each sidecar processed:

### Aggregated Parquet File

```
{input_stem}-aggregated[-densified].{sidecar_suffix}.parquet
```

**Index:** `healpix_id`

**Columns:**
- `{column}_{agg}`: Statistic for each value column × aggregation function
- `n_sources`: Observation count per cell

**Example:** `sample_50k-aggregated.cell-healpix_assignment-fuzzy_nside-32_order-nested.parquet`

### Metadata JSON File

```
{output_stem}.meta.json
```

**Contains:**
- Processing stage, timestamp
- Aggregation parameters (columns, functions, filters)
- Backend used (DuckDB, Dask)
- Sidecar metadata reference

## Examples

### Basic Aggregation

```bash
healpyxel_aggregate \
  --input observations.parquet \
  --sidecar-index 0 \
  --aggregate \
  --columns r1050 \
  --aggs mean median std mad robust_std
```

### Batch Process All Sidecars

```bash
healpyxel_aggregate \
  --input sample_50k.parquet \
  --sidecar-dir output/ \
  --sidecar-index all \
  --aggregate \
  --columns r750 r950 \
  --aggs median robust_std \
  --min-count 3 \
  --densify \
  -y
```

### With Quality Filtering

```bash
healpyxel_aggregate \
  --input observations.parquet \
  --sidecar-index 0 \
  --aggregate \
  --columns reflectance \
  --filter "qa_flag > 0.8 and solar_zenith < 80" \
  --aggs median robust_std \
  --min-count 5
```

### Inspect Before Aggregating

```bash
# Check input schema
healpyxel_aggregate -i observations.parquet --schema

# List available sidecars
healpyxel_aggregate -i observations.parquet --list-sidecars --stats

# Preview aggregation plan
healpyxel_aggregate -i observations.parquet --sidecar-index 0 \
  --aggregate --columns r750 --aggs median --dry-run
```

### Large Dataset with Dask

```bash
healpyxel_aggregate \
  --input large_dataset.parquet \
  --sidecar-index 0 \
  --aggregate \
  --columns value \
  --use-dask \
  --dask-npartitions 16 \
  --min-count 10
```

## Performance Tips

<Tip>
**For wide tables:** Use DuckDB to load only needed columns:

```bash
--use-duckdb --columns r750 r950  # Only loads these columns
```
</Tip>

<Tip>
**For parallel processing:** Tune Dask partitions to CPU count:

```bash
--use-dask --dask-npartitions 8  # Match available cores
```
</Tip>

<Tip>
**For quality control:** Filter early to reduce memory:

```bash
--filter "quality > 0.5" --min-count 3
```
</Tip>

## Troubleshooting

### Error: "No matches found between sidecar and data"

**Cause:** `source_id` mismatch between sidecar and input file.

**Solution:** Verify sidecar was generated from the same input file:
```bash
--list-sidecars --stats  # Check metadata
```

### Error: "Columns not found in input data"

**Cause:** Specified `--columns` don't exist in input file.

**Solution:** Check schema first:
```bash
--schema  # List available columns
```

### Warning: "X source_ids in sidecar not found in original data"

**Reason:** Filtered or modified input file since sidecar generation.

**Impact:** Missing observations are skipped (non-fatal).
